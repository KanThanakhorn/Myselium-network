# 🧪 คู่มือการทดสอบระบบด้วยตนเอง (Manual Testing Guide)
**Mycelium Forest Guard — IoT Sensors Network**

คู่มือฉบับนี้อธิบายลำดับขั้นตอนการทดสอบฟังก์ชันสำคัญของระบบเพื่อตรวจสอบการทำงานของ อัลกอริทึมการเลือกเส้นทาง (Routing Algorithm), ระบบกู้ภัยตัวเองของโครงข่าย (Self-Healing), ระบบสิทธิ์เจ้าหน้าที่ (RBAC) และการแจ้งเตือนเหตุฉุกเฉินไฟป่า (Real-time Fire Alerts)

---

## 🛠️ ขั้นตอนที่ 1: การเตรียมความพร้อมของระบบ (System Startup)

เปิดหน้าต่าง Command Line / Terminal เพื่อรันเซอร์วิสหลัก 3 ส่วนดังต่อไปนี้:

1. **ฝั่งเซิร์ฟเวอร์หลัก (Backend):**
   ```bash
   # รันที่ root directory ของโปรเจกต์
   npm run dev
   ```
2. **ฝั่งหน้าแอปพลิเคชันแดชบอร์ด (Frontend):**
   ```bash
   cd frontend
   npm run dev
   ```
   *เปิดเว็บเบราว์เซอร์เข้าไปที่: [http://localhost:5173](http://localhost:5173)*
3. **โปรแกรมจำลองข้อมูลบอร์ดเซนเซอร์ (IoT Simulator):**
   ```bash
   # เปิดรันที่ root directory ของโปรเจกต์
   npm run simulator
   ```

---

## 🔑 ขั้นตอนที่ 2: ทดสอบระบบเข้าสู่ระบบและการควบคุมสิทธิ์ (Authentication & RBAC)

**วัตถุประสงค์:** ตรวจสอบว่าระบบสามารถล็อกอินและจำกัดสิทธิ์การปรับแต่งเซนเซอร์ได้ถูกต้องตามบทบาทของเจ้าหน้าที่

1. **การตรวจสอบในฐานะบุคคลทั่วไป (Guest):**
   * ไปที่แท็บ **"Network Health"** บนแดชบอร์ด
   * เลือกคลิกที่จุดเซนเซอร์ใดก็ได้ (เช่น `node-01` หรือ `node-03`)
   * ตรวจสอบแผงควบคุมด้านขวาล่าง: ปุ่ม **"Recalibrate"** และ **"Deactivate / Activate"** จะต้องถูกล็อกเอาไว้ พร้อมข้อความแจ้งเตือน 🔒 *“Log in as Ranger or Administrator...”*
2. **การเข้าสู่ระบบในฐานะเจ้าหน้าที่ (Ranger):**
   * กดปุ่ม **"Sign In"** ที่มุมบนขวาของแดชบอร์ด
   * กรอกอีเมล: `ranger@mycelium.org` และรหัสผ่าน: `ranger123`
   * คลิกเซนเซอร์อีกครั้ง: ปุ่มการทำงาน **"Recalibrate"** และ **"Deactivate / Activate"** จะต้องเปิดให้สามารถใช้งานได้ตามปกติ

---

## 🕸️ ขั้นตอนที่ 3: ทดสอบการคำนวณเส้นทางและน้ำหนักเครือข่ายแบบเรียลไทม์ (Dynamic Routing & Weighting)

**วัตถุประสงค์:** พิสูจน์ว่าระบบคำนวณเส้นทางวิทยุที่ดีที่สุดและอัปเดตแบบ Dynamic ตามเวลาจริงเมื่อสภาวะแวดล้อมหรือสัญญาณวิทยุเปลี่ยนไป

1. สังเกตที่แผนที่กราฟวงกลมเซนเซอร์ (Topology Mesh Map) ในแท็บ **Network Health**:
   * เส้นเชื่อมระหว่างจุดต่าง ๆ จะต้องมีวงกลมสีเขียวเคลื่อนไหวตลอดเวลา เพื่อบอกทิศทางการส่งข้อมูล (Packet Flow) ไปยังโหนดปลายทาง
2. คลิกเลือกโหนดที่ทำงานอยู่ (เช่น `node-03`):
   * ตรวจสอบข้อมูลในหัวข้อ **"Mycelium Routing Engine"** บนการ์ดด้านขวา
   * ระบบจะแสดงค่า **Decision Weight (ค่าน้ำหนักการเลือกเส้นทาง)** เป็นเลขทศนิยม (เช่น `0.740`) 
   * ค่าน้ำหนักนี้เกิดจากการประมวลผลสดบนเซิร์ฟเวอร์ด้วยสูตรคำนวณพลังงานแบตเตอรี่ที่เหลืออยู่รวมกับคุณภาพสัญญาณวิทยุ (RSSI)

---

## 🩹 ขั้นตอนที่ 4: ทดสอบการเยียวยาตัวเองของโครงข่ายเมื่ออุปกรณ์ล่ม (Self-Healing Recovery)

**วัตถุประสงค์:** ตรวจสอบว่าระบบสามารถตรวจจับโหนดที่แบตเตอรี่หมดหรือออฟไลน์ และเปลี่ยนเส้นทางการส่งข้อมูลหลบเลี่ยงโหนดนั้นทันทีเพื่อไม่ให้สัญญาณหลักขาดหาย

1. เลือกคลิกเซนเซอร์หลักตัวหนึ่งที่มีหน้าที่ส่งต่อสัญญาณ (เช่น `node-03`)
2. กดปุ่ม **"Deactivate"** (หรือตัวจำลองสัญญาณรายงานว่าแบตเตอรี่ของโหนดนั้นลดเหลือ 0%) เพื่อปิดการทำงานของอุปกรณ์:
   * **สแกนภาพแดชบอร์ดทันที:** จุดของโหนดนั้นจะเปลี่ยนจากสีเขียวเป็นสีเทา (Offline)
   * **ตรวจสอบการฟื้นตัว:** เส้นเชื่อมโยงเส้นทางจะทำการคำนวณใหม่และหักเลี้ยวอ้อมข้ามจุดที่ดับนั้นไปยังโหนดสำรองตัวอื่นโดยอัตโนมัติ เพื่อประคองการส่งข้อมูลไปยังเซิร์ฟเวอร์ปลายทาง
   * ตรวจสอบแท็บแจ้งเตือนด้านล่างสุดของแดชบอร์ด: ระบบจะบันทึกสถานะเหตุการณ์ **"self-healing"** ของระบบโครงข่ายระบุเวลาและช่องทางเชื่อมโยงใหม่โดยละเอียด

---

## 🚨 ขั้นตอนที่ 5: ทดสอบการตรวจจับไฟป่าและการจัดการแจ้งเตือนฉุกเฉิน (Emergency Fire Alerts)

**วัตถุประสงค์:** ทดสอบระบบแจ้งเตือนภัยแบบเร่งด่วนเมื่อตัวตรวจจับเจอกลุ่มควันหนาแน่นสูงหรือความร้อนวิกฤต

1. **การสร้างสัญญาณจำลองไฟไหม้:**
   * สังเกตโปรแกรม IoT Simulator ที่ทำงานอยู่ในหน้า Terminal 
   * เมื่อโปรแกรมสุ่มจำลองไฟป่า (สุ่มเจอโอกาส 1% หรือสามารถคลิกที่ปุ่มทดสอบจำลองส่งสปายอุณหภูมิวิกฤตส่งไปที่ `/api/nodes/node-03/telemetry`)
2. **การทำงานของแจ้งเตือน:**
   * บนหน้าจอแดชบอร์ดจะกะพริบแบนเนอร์แจ้งเตือนไฟสีแดงขนาดใหญ่ด้านบนสุดของหน้าเว็บ
   * จะต้องมีเสียงสัญญาณเตือนภัย (Siren) ดังขึ้นผ่านทางลำโพงคอมพิวเตอร์ของคุณ
   * เมนูกล่องเตือนภัยที่มุมขวาบนจะขึ้นแจ้งเหตุสีแดงฉุกเฉิน (Critical Alert)
3. **การรับรองและเขียนรายงานการแก้ไข (Acknowledge & Resolve):**
   * กดเข้าไปที่เมนูแจ้งเตือนสีแดง หรือไปที่แท็บ **"Alerts"**
   * กดปุ่ม **"Acknowledge"** เพื่อยืนยันว่าเจ้าหน้าที่ทราบเรื่องแล้ว (เสียงเตือนภัยจะเงียบลง)
   * กดปุ่ม **"Resolve"** เพื่อกรอกบันทึกการเข้าระงับเหตุของเจ้าหน้าที่ (เช่นระบุว่า *"ส่งเฮลิคอปเตอร์เข้าไปโปรยน้ำระงับไฟป่าแล้ว สภาพพื้นที่ควบคุมได้เรียบร้อย"*) เมื่อกดบันทึกแล้ว รายงานนี้จะย้ายเข้าไปสู่ประวัติเหตุการณ์ของอุทยานประวัติศาสตร์อย่างถาวร

---

## 📊 ขั้นตอนที่ 6: การรันจำลองเปรียบเทียบในสภาพแวดล้อมจำลอง (Simulation Scripts Execution)

**วัตถุประสงค์:** รันโปรแกรมเปรียบเทียบเชิงสถิติระหว่างการเดินเส้นทางข้อมูลแบบสถิตทั่วไป (Shortest-Path) กับระบบใยรา Mycelium

1. เปิด Terminal ใหม่แล้วเข้าไปยังโฟลเดอร์ของโปรเจกต์
2. รันโปรแกรมคำนวณเปรียบเทียบอายุการใช้งานของเครือข่ายเซนเซอร์:
   ```bash
   python3 simulation/mycelium_routing.py
   ```
   *ตรวจเช็กผลลัพธ์บนจอ: ระบบต้องแสดงตารางสรุปอายุการใช้งาน โดยสัดส่วนการประหยัดไฟเครือข่ายของ Mycelium จะต้องปรับปรุงประสิทธิภาพขึ้น **มากกว่า 20% ถึง 30% ขึ้นไป** เมื่อเทียบกับเครือข่ายแบบเดิม*
3. รันตัวประเมินโครงสร้างเครือข่ายไร้สายด้วย NS-3 Config Simulator:
   ```bash
   python3 simulation/ns3_config.py
   ```
   *ตรวจสอบระดับประสิทธิภาพความเสถียรของการส่งข้อมูล (Packet Delivery Ratio) และระดับความหน่วงในการตอบสนอง (Latency)*
